# Website v2 Development Workflow

## Quick Start — Live Preview

```bash
cd C:/Projects/Westside-Website-v2
npm start
# or: browser-sync start --config bs-config.js
```

Opens HTTPS server at:
- **Desktop:** https://localhost:3000
- **Phone/other devices:** https://<your-ip>:3000 (BrowserSync prints the External URL)
- **BrowserSync UI:** http://localhost:3001 (toggle sync options, see connected devices)

CSS changes inject instantly (no reload). HTML/JS changes trigger full reload.

## Mobile Testing Options (best → simplest)

### 1. Real Phone — Chrome Remote Debugging (best fidelity)
```bash
# Connect phone via USB (enable USB Debugging in Developer Options)
adb devices                              # verify phone shows up
adb reverse tcp:3000 tcp:3000            # forward port to phone
# On phone: open Chrome → chrome://inspect → browse to https://localhost:3000
# On desktop: chrome://inspect → see phone's tabs, inspect remotely
```

### 2. Real Phone — Same Network (easiest, no USB)
Just open the BrowserSync External URL on your phone's browser. With Tailscale, this works from any network.

### 3. Real Phone — Cloudflare Tunnel (share with anyone, anywhere)
```bash
# In a separate terminal:
pwsh -NoProfile -Command 'cloudflared tunnel --url https://localhost:3000'
# Gives you a *.trycloudflare.com URL — works on any phone, any network
```

### 4. Browser DevTools Emulation (quick checks)
Chrome/Edge: F12 → Toggle Device Toolbar (Ctrl+Shift+M) → pick a device preset.
Firefox: F12 → Responsive Design Mode (Ctrl+Shift+M).
**Note:** Emulation approximates mobile but is NOT a substitute for real-device testing.

## Linting & Auditing

```bash
npm run lint:html     # HTML validation (offline, W3C-like rules)
npm run lint:css      # CSS linting (needs stylelint-config-standard)
npm run lint:js       # JS linting
npm run lint          # all three

npm run format:check  # check formatting (Prettier)
npm run format        # auto-format all HTML/CSS/JS

npm run audit:a11y    # accessibility test (WCAG2AA)
npm run audit:links   # broken link check (recursive)
npm run audit:lighthouse  # full Lighthouse audit
```

## HTTPS Notes
- Certs are in `.certs/` (gitignored), generated by `mkcert`
- Chrome/Edge trust them automatically
- Firefox: set `security.enterprise_roots.enabled = true` in `about:config`
- To regenerate: `mkcert -cert-file .certs/localhost.pem -key-file .certs/localhost-key.pem localhost 127.0.0.1 ::1`

## Playwright (automated cross-browser testing)
```bash
npx playwright codegen --device="iPhone 13" https://localhost:3000   # record test
npx playwright test --ui                                              # run tests with UI
npx playwright test                                                   # headless run
```
